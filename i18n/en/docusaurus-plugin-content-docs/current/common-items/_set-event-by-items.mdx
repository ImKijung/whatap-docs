## Setting the events by item

You can set the following events.

- <Cmdname sid="event_apm_cpu" className="uitext" />, <Cmdname sid="event_apm_mem" className="uitext" />, <Cmdname sid="event_apm_disk" className="uitext" />

  A notification is sent when the usages of each item reach the threshold set in <Cmdname sid="condition" className="uitext" /><span class='uitext'>(%)</span>.

- <Cmdname sid="event_apm_acttx" className="uitext" />

  If the number of active transactions continues to exceed the threshold set in <Cmdname sid="condition" className="uitext" /><span clas='uitext'>(</span><Cmdname sid="count" className="uitext" /><span clas='uitext'>)</span>, the corresponding alert is sent. The default value of occurrence is 100.

- <Cmdname sid="event_apm_errtx" className="uitext" />

  It indicates a transaction that has not been performed normally. If the error count continues to exceed the threshold set in <Cmdname sid="condition" className="uitext" /><span clas='uitext'>(</span><Cmdname sid="count" className="uitext" /><span clas='uitext'>)</span>, the corresponding alert is sent. The collection cycle is 5 seconds.

- <Cmdname sid="event_apm_resp" className="uitext" />

  Check with the transaction end time. For example, if the <Cmdname sid="count" className="uitext" /> of transactions that are above the <Cmdname sid="response_time" className="uitext" /> threshold (**10** seconds) exceeds **10**, and this state <Cmdname sid="continue" className="uitext" /> for **5** seconds, the corresponding alert is sent.
