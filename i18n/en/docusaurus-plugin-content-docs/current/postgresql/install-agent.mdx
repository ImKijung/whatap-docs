Assign additional roles depending on the features and operation environment.

- PostgreSQL 10 or later

  ```sql title="SQL"
  create user DB_User with password 'DB Password'; 
  grant pg_read_all_stats to DB_User;
  ```

- Below PostgreSQL 10

  ```sql title="SQL"
  create user DB_User with password 'DB Password';
  alter user DB_User with superuser;
  ```

- In case of using the kill session

  Database monitoring provides the <UI>Kill Session</UI> function that allows you to kill an active session. To use this function, set the role as follows:

  ```sql title="SQL"
  alter user DB_User with superuser;
  ```

- In case of using the kill session in the AWS RDS environment

  ```sql title="SQL"
  grant rds_superuser to DB_User;
  ```

- In case of collecting wal_count

  ```sql title="SQL"
  grant execute on function pg_ls_dir(text) to DB_User;  # In case of use of EDB
  alter user DB_User with superuser;
  ```

- In case of collecting the cache

  ```sql title="SQL"
  grant select on pg_backend_memory_contexts to DB_User;
  grant execute on function pg_get_backend_memory_contexts to DB_User;
  ```

- In case of collecting the table size

  ```sql title="SQL"
  grant usage on schema {schema_name} to DB_User;
  ```

- In case of collecting the object information (<Cmdname sid="side_analysis" className="uitext" /> > <Cmdname sid="side_dbObjectInfo" className="uitext" />)

  ```sql title="SQL"
  grant select on all tables in schema {schema_name} to DB_User;
  ```
