March 25, 2024

## Common

<Status>Changed</Status> Deleted the <code>xos2</code> metric from the <code>cpu</code> metrics and changed the <code>xos</code> metric to percentage.

## MongoDB

<Status>Feature</Status> Added MongoDB <code class="my">V2</code>

- Category name unification

  - <code>agentinfo</code> → <code>db_agentinfo</code>
  - <code>session</code> → <code>db_mongo_active_session</code>
  - <code>stat</code> → <code>db_mongodb_counter</code>
  - <code>dbsize</code> → <code>db_dbsize</code>

- Basic metrics simplification

  If <code>v2=true</code> is set, the system collects and saves <code>opcounters</code>, <code>globalLock</code>, <code>extra_info</code>, <code>connections</code>, <code>network</code>, <code>mem</code>, <code>opLatencies</code>, and <code>asserts</code> metrics among <code>serverStatus</code> values.

  :::note

  <code>v2=false</code> is the default value. As in v1, the <code>serverStatus</code> results are all collected.

  :::

- Additional metric options: If any outputs (<code>category=network, repl, indexStats</code>, etc.) of serverStatus are additionally set, additional collection is possible.

- Addition of database size collection: Collects <code>index size</code>, <code>storage size</code>, and <code>total size</code>.

- Collection size collection (new): Collects <code>storage size</code>, <code>index size</code>, and <code>total size</code>.

- The xos metrics (<code>xos_cpu</code>, <code>xos_mem</code>, etc.) that were previously indicated with underscores, are also sent along with values including parentheses for product uniformity.

## Redis

<Status>Feature</Status> Added Redis <code class="my">V2</code>

- Category name unification

  - <code>agentinfo</code> → <code>db_agentinfo</code>
  - <code>session</code> → <code>db_redis_active_session</code>
  - <code>stat</code> → <code>db_redis_couter</code>

- Added the <code>db_redis_clusterinfo</code>, <code>db_redis_clusternodes</code>, <code>db_redis_master</code>, and <code>db_redis_slave</code> categories.

- The xos metrics (<code>xos_cpu</code>, <code>xos_mem</code>, etc.) that were previously indicated with underscores, are also sent along with values including parentheses for product uniformity.
