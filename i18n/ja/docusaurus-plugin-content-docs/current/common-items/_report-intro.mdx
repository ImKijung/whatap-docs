ホーム画面 > プロジェクト選択 > **_統計・レポート_** > **_レポート_**

WhaTapモニタリングサービスのホーム画面でプロジェクトを選択し **_プロジェクトメニュー_**の **_統計/レポート_** > **_レポート_**を選択してください。 

レポートは、個々のプロジェクトのサービスの使用状況と障害の発生記録を報告する文書です。サービスモニタリング担当者は、レポートを通じて関連部署の担当者とステータスを共有します。

モニタリングデータ分析は、サービスの改善方向を定める指標となるため、極めて重要です。ただし、複数のダッシュボードからデータを収集して文書化するには煩雑な場合があります。  

WhaTapの**_レポート_**メニューから、**レポート作成業務の自動化**をサポートします。レポートメニューから、週次レポートを作成したり、設定した時間にレポートを共有したり、さまざまな形式で管理することができます。

:::tip

**レポート**をダウンロードする場合、印刷またはメール送信の予約をスケジュールする場合は、そのボタンをクリックしてください。ダウンロードはhtml形式で入手可能です。

:::

## レポートと統合レポートの違い

**_レポート_**は、一つのプロジェクトに対するレポートを作成するメニューです。従って、プロジェクトを選択した後、左部の**_統計／レポート_**メニューからアクセスできます。

**_統合レポート_**は、複数のプロジェクトのレポートを生成するメニューです。プロジェクトを選択することなく、基本メニューからアクセスできます。**_統合レポート_**の詳細については、[次の文書](integrated-report)で確認できます。

## メール送信予約

定期的にレポートをメールで受け取りたい場合は、**_レポートのメール送信予約_**を選択してください。出社直後に実行していた様々なサービス点検に関する手続きをメール確認に置き換えることができます。
