## 項目別イベント設定

次のイベント項目を設定できます。

-   **\*サーバーCPU**、**\*サーバーメモリ**、**_サーバーディスク_**

    各項目の使用量で設定した**_条件(%)_**に設定したしきい値に達すると通知が送信されます。

-   **_アクティブトランザクション_**

    アクティブトランザクションの発生件数が**_条件(件数)_**に設定したしきい値を超え続ける場合は、通知を送信します。発生件数のデフォルト値は100個です。

-   **_トランザクションエラー_**

    正常に実行されてかったトランザクションを意味します。エラー件数が**_条件(件数)_**に設定したしきい値を超えている場合は通知を送信します。収集間隔は5秒です。

-   **_トランザクションレスポンス_**

    トランザクションが終了した時間を確認します。例えば、**10**秒の**_レスポンス時間_**しきい値を超えるトランザクションの**_件数_**が**10**個を超え、この状態が**5**秒間**_持続_**すると通知を送信します。

-   **_ヒートマップ横ライン_**

    ヒートマップのレスポンス時間分布で、**_レスポンス時間_**が**5**秒以上のトランザクションのレスポンス時間に連続して、<sup>\*</sup> **5** **_件数_**が発生すると通知します。

    :::note

    -   \* X軸(5秒) 単位の分布は実線で表示されます。 (途中で区切れがある場合は適用されません)
    -   これは未対応の通知設定機能です。**_ヒートマップパターン_**通知を使用することをお勧めします。

    :::

-   **_ヒートマップ縦ライン_**

    ヒートマップのレスポンス時間分布からX軸に１つの間隔(5秒)に縦線<sup>\*</sup>が表示され、その時間(1回)5秒**_続く_**と通知を送信します。

    :::note

    -   \* 縦線現象：**_最小時間(秒)_** 2.5秒から80秒を表す合計100間隔のうち、3**30~50**間隔をマーキングする場合は、<span class='slow'>アラート通知</span>が発生し、間隔が![指示線2](/img/number-02.png)**50**を超えるマーキングされた場合、<span class='vslow'>危険通知</span>が発生します。
    -   これは未対応の通知設定機能です。**_ヒートマップパターン_**通知を使用することをお勧めします。

    :::
