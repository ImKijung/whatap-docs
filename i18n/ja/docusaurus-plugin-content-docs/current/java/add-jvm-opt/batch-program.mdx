バッチアプリケーションで、WhaTapエージェントを適用する方法です。 

バッチアプリケーションは、WASとは異なり、アプリケーションが実行と中断を繰り返す特長があります。したがって、一般的なウェブアプリケーションとは異なるモニタリングを行う必要があります。

![Batch Application](/img/batch-app-intro.png)

## バッチジョブ(Batch Job) のインストール

バッチジョブ(Batch Job)を1つのトランザクションの観点からモニタリングする必要があります。そのため、WhaTapのJavaエージェントは、Jobプロセス用のJob Agentと、この情報をサーバーに中継するためのHost Agentに分離した2つのエージェントをインストールする必要があります。

```bash title='Batch Job Agent'
java -javaagent:/full/path/to/whatap.agent.batch-X.Y.Z.jar -jar {batch job jar}
```

```bash title='Batch Host Agent'
java -jar whatap.agent.host-X.Y.Z.jar
```

:::note

Batch Job AgentとBatch Host Agentは、Batch Jobを実行するホストと一緒にインストールしてください。

:::

## Time Limitの設定

Batch Job AgentとBatch Host Agentの間はUDPを使用して通信します。

![Batch Job UDP](/img/batch-app-udp.png)

バッチジョブは通常、長時間行われます。数十秒以上掛かる場合もあります。したがって、バッチジョブごとに、`time_limit`の値は異なる場合があります。バッチジョブが実行される予定処理時間を`time_limit`に設定してください。

```ini title=whatap.conf
time_limit=300000
warning_time=time_limit's 70%
step_interval=5000
```

## アクティブイコライザーカラーの設定

バッチジョブの実行時間が、`time_limit`の70％に達すると黄色で表示し、`time_limit`を超えると赤色で表示します。

```ini title='whatap.conf' {2,4} showLineNumbers
## 終了予定時間の70％
yellow_time = time_limit * 0.7;
## 終了予定時間を超過
red_time = time_limit;
```

<ImgLang img='batch-app-timelimit.png' desc='Batch Job UDP' />

:::note

サーバー運用者は、バッチジョブの遅延をリアルタイムで確認するには、`time_limit`値を適切に設定してください。

:::
