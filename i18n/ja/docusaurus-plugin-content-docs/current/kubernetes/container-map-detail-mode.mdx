**_コンテナマップ_**メニューで選択した対象に対する詳細情報を、**_詳細を見る_**画面で確認できます。

1.  **_コンテナマップ_**メニュー画面で照会したい特定のブロック(コンテナまたはPod)を選択してください。 

2.  画面右側に次のように**要約を見る**パネルが表示されます。 

  ![コンテナマップ - 要約を見る](/img/k8s-container-sum.png)

1.  **要約を見る**パネルの右上の![詳細を見るアイコン](/img/ico-detail-mode.svg) **_詳細を見る_**アイコンを選択すると、より詳細な情報を確認できます。

## 基本画面ガイド

コンテナマップの**詳細を見る**画面は、次のように構成されています。基本情報は、カードと一覧形式で確認できます。重要モニタリングデータを照会できるように**_トレース_**、**_メトリクス_**、**_ログ_**、**_イベント_**タブを提供します。 

![詳細を見る画面](/img/k8s-container-detail.png)

-   ![number 1](/img/number-01.png) **要約情報カード** (コンテナまたはPod)

    選択対象のグループ情報を確認できます。対象の追加選択時、![number 4](/img/number-04.png)領域で選択した対象の指標を確認できます。コンテナマップで選択したカードと同じビューを持っています。 詳細については、[次の文書](container-map)を参考にしてください。

-   ![指示線2](/img/number-02.png) **要約情報一覧**

    選択対象のグループ情報を一覧形式で確認できます。対象の追加選択時、![number 4](/img/number-04.png)領域で選択した対象に対する指標を確認することができます。 

    -   ![もっと見るアイコン](/img/right-arrow.svg) **_もっと見るアイコンを選択すると、_**Node**_、_**Namespace**_、_**Deployment**_、_**ReplicaSet**_、_**Pod**_、_**ContainerName**_、_**ContainerId**_、_**Image**_、_**Agent\*\*\*などの指標情報を照会することができます。

        -   **_Node_**：ノード詳細、マスターメタ情報

        -   **_Deployment_**：マスターメタ情報

        -   **_ReplicaSet_**：マスターメタ情報

        -   **_Pod_**：マスターメタ情報

        -   **_ContainerId_**：コンテナ情報

    -   ![もっと見るアイコン](/img/ico-more.svg) **_詳細照会_**：各指標案内の右側にある「詳細照会アイコン」を選択すると、詳細またはマスターメタ情報を照会することができます。

-   ![指示線4](/img/number-04.png) **_トレース_**、**_メトリックス_**、**_ログ_**、**_イベント_** 

    -   **_トレース_** 

        **_トレース_**タブでは、コンテナでサービス中のアプリケーション情報を読み込み、当該アプリケーションのトランザクション状況をモニタリングできます。**_ダッシュボード_**ビューと**_トランザクション一覧_** ビューを提供します。詳細については、[次の文書](container-map-trace)を参考にしてください。

    -   **_メトリクス_**

        **_メトリクス_**タブでは、モニタリングに有用な指標を選別してチャートに表示します。選択したコンテナまたはPodのリソースが適切に設定されているかを把握できます。詳細については、[次の文書](container-map-metrics)を参考にしてください。

    -   **_ログ_** 

        **_ログ_**タブでは、Kubernetesで発生するログを照会できます。コンテナを終了すると、関連ログまですべて消えます。過去に発生したログは、**_ログ検索_**オプションで確認してください。詳細については、[次の文書](container-map-log)を参考にしてください。

    -   **_イベント_** 

        **_イベント_**タブでは、Kubernetesから発生する**_Kubernates_**イベントとユーザーが設定したイベント発生条件によって生成された**_WhaTap_**イベントを照会することができます。 詳細については、[次の文書](container-map-event)を参考にしてください。
