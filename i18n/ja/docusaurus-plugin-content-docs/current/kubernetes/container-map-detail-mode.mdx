**_コンテナマップ_**メニューで選択した対象の詳細情報を、**_詳細表示_**画面で確認できます。

1.  **_コンテナマップ_**メニュー画面で表示する特定のブロック(コンテナまたはPod)を選択してください。 

2.  画面の右側に、次のように**概要ビュー**パネルが表示されます。 

    <ImgLang img='k8s-container-sum.png' desc='コンテナマップ - 概要を表示' />

3.  **概要を表示**パネル右上の![詳細を見るアイコン](/img/ico-detail-mode.svg) **_詳細を見る_**アイコンを選択すると、詳細な情報が確認できます。

## 基本画面ガイド

コンテナマップの**詳細ビュー**画面は、次のように構成されています。基本情報は、カードと一覧形式で確認できます。重要なモニタリングデータを表示できるように**_トレース_**、**_メトリクス_**、**_ログ_**、**_イベント_**タブを提供します。 

<ImgLang img='k8s-container-detail.png' desc='詳細画面' />

-   ![number 1](/img/number-01.png) **概要情報カード** (コンテナまたはPod)

    選択した対象のグループ情報を確認できます。対象の宛先追加を選択時、![number 4](/img/number-04.png)領域で選択した対象の指標を確認できます。コンテナマップで選択したカードと同じビューを持っています。 詳細については、[次の文書](container-map)を参照してください。

    :::note

    集計情報カード参照時にブロック数が多い場合、次のように表記方法を変更します。すべてのブロック、<span class='normal'>正常</span>、<span class='slow'>アラート</span>、<span class='vslow'>警告</span>状態ブロック数を確認して比較することができます。

    <ImgLang img='k8s-container-card-block.png' desc='ブロック数が多い場合は、集計情報カード' />

    :::

-   ![指示線2](/img/number-02.png) **概要情報一覧**

    選択した対象のグループ情報を一覧形式で確認できます。対象の追加を選択時、![number 4](/img/number-04.png)領域で選択した対象の指標を確認することができます。 

    -   一覧で![指示線3](/img/number-03.png)選択した対象の左側の![もっと見るアイコン](/img/right-arrow.svg)**_もっと見る_** アイコンを選択すると**_Node_**、**_Namespace_**、**_Deployment_**、**_ReplicaSet_**、**_Pod_**、**_ContainerName_**、**_ContainerId_**、**_Image_**、**_Agent_**などのメトリクス情報を照会することができます。

        -   **_Node_**：ノード詳細、マスターメタ情報

        -   **_Deployment_**：マスターメタ情報

        -   **_ReplicaSet_**：マスターメタ情報

        -   **_Pod_**：マスターメタ情報

        -   **_ContainerId_**：コンテナ情報

    -   ![詳細アイコン](/img/ico-more.svg) **_詳細表示_**：各指標ガイドの右側にある「詳細表示アイコン」を選択すると、詳細またはマスターメタ情報を表示できます。

-   ![指示線4](/img/number-04.png)**トレース**、**メトリクス**、**ログ**、**イベント**、**呼び出し情報**

    -   **_トレース_** 

        **_トレース_**タブを使用して、クバネティスコンテナ上で実行中のアプリケーションのトランザクション詳細をモニタリングできます。**_ダッシュボード_**ビューと**_トランザクション一覧_**ビューを提供します。詳細については[次の文書](container-map-trace)を参照してください。

    -   **_メトリクス_**

        **_メトリクス_**タブを使用して、選択されたコンテナまたはPodの主要なメトリクス情報とリソース使用パターンを確認できます。これにより、リソース関連の問題をすばやく識別し、最適化ポイントを見つけることができます。詳しい内容は[次の文書](container-map-metrics)を参照してください。

    -   **_ログ_** 

        **_ログ_**タブを通じてクバネティスコンテナ及びコンテナアプリケーションで発生するログを照会することができます。ログ機能は、コンテナのリアルタイムログ照会機能と特定の時点での過去ログ検索機能に分けられます。詳細については[次の文書](container-map-log)を参照してください。

    -   **_イベント_** 

        **_イベント_**タブを介してクバネティスクラスターで発生する**_Kubernetes_**イベントとユーザー自身で設定した条件による**_WhaTap_**イベントを照会することができます。詳細については[次の文書](container-map-event)を参照してください。

    -   **_呼び出し情報_**

        **_呼び出し情報_**タブを通じてクバネティスコンテナ上で実行中のアプリケーションの呼び出しフローと外部呼び出し対象の状態を確認することができます。詳細については[次の文書](container-map-call-information)を参照してください。
