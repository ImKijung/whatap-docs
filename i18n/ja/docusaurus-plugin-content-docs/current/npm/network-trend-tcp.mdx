ホーム画面 > プロジェクト選択 > **_ダッシュボード_** > **_ネットワーク推移(TCP)_**

![ネットワーク推移(TCP)](/img/npm-dashboard-network-trend.png)

**_ネットワークトポロジー_**メニューは、ネットワーク構成情報に対する視覚化を主な目的としていることに対し、**_ネットワーク推移(TCP)_**メニューは、数値で表示される性能情報を一つの画面で高い密度で確認できます。 時間に応じた性能情報をリアルタイムで確認することができ、過去時点の性能情報を確認して問題点を分析する資料として活用できます。

### ノード検索

画面右上の検索入力ボックスを通じてチャートに表示されるノードをフィルタリングできます。 検索語にマッチしない項目は、チャート上で表示されません。

:::note

この機能は、特定のタグに関連する通信のみを確認する必要がある場合に役立ちます。

:::

## Top 5チャート{#topn}

![Top 5 Chart](/img/npm-network-trend-top5.png)

ネットワークの通信障害を意味する主要指標である**jitter**と**latency**を上位5位までをチャートとして表示します。

## 再送信およびパケット損失指標

![再送信およびパケット損失指標](/img/npm-network-rc-lc-chart.png)

特定の通信区間で再送信が行われた回数をチャートとして表示します。 再送信の原因となり得るパケット損失回数と連携して、再送信状況に対するパケット損失有無を確認することができます。

## 遅延率指標

![遅延率指標](/img/npm-network-trend-jitter-latency.png)

遅延率と遅延率の変化に基づいてネットワークの安定性を評価できます。

## データ処理量指標

![データ処理量指標](/img/npm-network-trend-data-process.png)

データ処理量(bpsまたはpps) 指標を送受信の観点から分けてチャートで表示します。 

## セッション情報

![セッション情報テーブル](/img/npm-network-trend-sessions.png)

エージェントを通じて収集したセッションデータをテーブル形式で表示します。 区間別にマージされたチャートを通じてマクロ的な問題状況を識別し、ミクロ的に問題を分析する必要がある状況で活用できます。
