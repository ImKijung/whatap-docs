機能と使用環境に応じて追加権限を付与してください。

- PostgreSQL 10バージョン以降

  ```sql title="SQL"
  create user DB_User with password 'DB Password'; 
  grant pg_read_all_stats to DB_User;
  ```

- PostgreSQL 10バージョン未満

  ```sql title="SQL"
  create user DB_User with password 'DB Password';
  alter user DB_User with superuser;
  ```

- Kill sessionを使用する場合

  データベースモニタリングでは、進行中のセッションを終了できる<UI>Kill Session</UI>機能を提供します。この機能を使用するには、次のように権限を設定してください。

  ```sql title="SQL"
  alter user DB_User with superuser;
  ```

- AWS RDS環境でKill session機能を使用する場合

  ```sql title="SQL"
  grant rds_superuser to DB_User;
  ```

- wal_countを収集する場合

  ```sql title="SQL"
  grant execute on function pg_ls_dir(text) to DB_User;  # EDB使用時
  alter user DB_User with superuser;
  ```

- cacheを収集する場合

  ```sql title="SQL"
  grant select on pg_backend_memory_contexts to DB_User;
  grant execute on function pg_get_backend_memory_contexts to DB_User;
  ```

- テーブルサイズを収集する場合

  ```sql title="SQL"
  grant usage on schema {schema_name} to DB_User;
  ```

- オブジェクト情報を照会する場合(<Cmdname sid="side_analysis" className="uitext" /> > <Cmdname sid="side_dbObjectInfo" className="uitext" />)

  ```sql title="SQL"
  grant select on all tables in schema {schema_name} to DB_User;
  ```
